## 2. 내 지식 수준 (배운 것)

### ✅ JavaScript 기초 (13개 완주!)
| # | 주제 | 핵심 |
|---|---|---|
| 1 | 변수와 상수 | `const`/`let`, 명령문은 undefined 메아리 |
| 2 | 자료형 | number/string/boolean/undefined/null/object, `typeof` |
| 3 | 함수 | 선언≠실행, `()`=실행버튼, `console.log`≠`return` |
| 4 | 함수 표현식 | 함수도 값, 호출 자리는 return값으로 대체 |
| 5 | 화살표 함수 | `=>`, 암묵적 반환 |
| 6 | 객체 | `{key:value}`, 변수는 객체를 "화살표(참조)"로 가리킴 |
| 7 | 참조에 의한 복사 | 객체=화살표만 복사(공유) / 원시값=값 복사(독립), `===`는 참조 비교 |
| 8 | 배열 | `[]`, 인덱스 0부터, 배열도 객체 |
| 9 | 배열 메서드 | `map`(변환)/`filter`(거르기)/`find`(첫 하나)/`forEach`(실행만) |
| 10 | 구조 분해 할당 | `const {a}=obj` / `const [a]=arr` |
| 11 | 옵셔널 체이닝 | `?.` — 앞이 null/undefined면 멈추고 undefined |
| 12 | Rest·Spread | `...` — 펼치기(spread)/모으기(rest) |
| 13 | 클로저 | 함수가 태어난 곳의 변수를 "주머니에 챙겨" 기억 |

### 🔵 React (입문 중)
- **useState**: `const [count, setCount] = useState(0)` 원리 이해 ✅
  - useState=함수, `[현재값, 바꾸는함수]` 배열 돌려줌
  - setCount = 값 바꾸기 + React에게 "다시 그려!" 알리기
  - 리렌더 = 컴포넌트 함수 통째 재실행
  - state는 React "주머니"에 저장 (= 클로저 원리!)
- **모듈**: `import`/`export`/`export default` (named=중괄호로 골라, default=대표 하나)
- **React 3대 특징**: 컴포넌트 기반 / 선언형(쉬운 업데이트) / Virtual DOM(빠른 업데이트)
- **브라우저 렌더링**: HTML→DOM, CSS→CSSOM → Render Tree → Layout → Painting
